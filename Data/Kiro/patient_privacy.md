# אבטחת מידע ופרטיות מטופלים

## סקירה כללית

מערכת ניהול המרפאה מיישמת פרוטוקולי אבטחה מתקדמים להגנה על מידע רפואי רגיש (PHI - Protected Health Information) בהתאם לתקני HIPAA ולתקנות הגנת הפרטיות המקומיות.

## עמידה בתקני HIPAA

### HIPAA Privacy Rule

המערכת עומדת בדרישות ה-Privacy Rule באמצעות:

- **הגבלת גישה למידע**: מימוש עקרון ה-Minimum Necessary - כל משתמש רואה רק את המידע הנדרש לתפקידו
- **תיעוד גישות**: רישום מלא של כל פעולות הצפייה, העריכה והמחיקה של נתוני מטופלים
- **הסכמות מטופל**: מנגנון דיגיטלי לקבלת הסכמות מפורשות לשימוש ושיתוף מידע רפואי
- **זכויות מטופל**: אפשרות למטופלים לצפות, להעתיק ולתקן את המידע הרפואי שלהם

### HIPAA Security Rule

#### אבטחה מנהלית (Administrative Safeguards)

- **ניהול סיכונים**: ביצוע הערכות סיכונים תקופתיות (Risk Assessment) לזיהוי איומים פוטנציאליים
- **הכשרת צוות**: תוכנית הדרכה שנתית חובה לכל העובדים בנושאי אבטחת מידע ופרטיות
- **תוכנית תגובה לאירועים**: נהלים מוגדרים לטיפול בהפרות אבטחה (Breach Response Plan)
- **ניהול גישות**: תהליך פורמלי למתן, שינוי ושלילת הרשאות גישה

#### אבטחה פיזית (Physical Safeguards)

- **בקרת גישה למתקן**: הגבלת גישה פיזית לשרתים ותחנות עבודה
- **אבטחת תחנות עבודה**: נעילה אוטומטית לאחר חוסר פעילות, מיקום מסכים הרחק מעיני הציבור
- **השמדת מדיה**: נהלים מאובטחים למחיקה והשמדה של התקני אחסון

#### אבטחה טכנית (Technical Safeguards)

- **בקרת גישה**: מימוש אימות דו-שלבי (2FA) חובה לכל המשתמשים
- **הצפנת נתונים**: 
  - Encryption at Rest: AES-256 להצפנת מסדי נתונים וקבצים
  - Encryption in Transit: TLS 1.3 לכל התקשורת הרשתית
- **ביקורת ומעקב**: לוגים מפורטים של כל פעולה במערכת עם שמירה למשך 7 שנים
- **אימות משתמשים**: 
  - סיסמאות חזקות (12+ תווים, מורכבות)
  - פקיעת סיסמה כל 90 יום
  - נעילת חשבון לאחר 5 ניסיונות כושלים

## ארכיטקטורת אבטחה

### שכבות הגנה (Defense in Depth)

```
┌─────────────────────────────────────┐
│   Web Application Firewall (WAF)   │
├─────────────────────────────────────┤
│   API Gateway + Rate Limiting       │
├─────────────────────────────────────┤
│   Application Layer Security        │
│   - Input Validation                │
│   - Output Encoding                 │
│   - CSRF Protection                 │
├─────────────────────────────────────┤
│   Database Layer                    │
│   - Encrypted at Rest               │
│   - Parameterized Queries           │
│   - Row-Level Security              │
├─────────────────────────────────────┤
│   Infrastructure Security           │
│   - Network Segmentation            │
│   - VPC Isolation                   │
└─────────────────────────────────────┘
```

### ניהול מפתחות הצפנה

- **Key Management Service (KMS)**: שימוש ב-AWS KMS או Azure Key Vault לניהול מפתחות
- **Key Rotation**: החלפת מפתחות הצפנה אוטומטית כל 90 יום
- **Separation of Duties**: הפרדה בין מפתחות לסביבות שונות (ייצור, פיתוח, בדיקות)

## בקרת גישה מבוססת תפקידים (RBAC)

### תפקידים מוגדרים

| תפקיד | הרשאות | הגבלות |
|-------|---------|---------|
| **מנהל מערכת** | ניהול משתמשים, תצורת מערכת, גישה ללוגים | אין גישה למידע רפואי |
| **רופא** | צפייה ועריכה של תיקי מטופלים משויכים, רישום אבחנות | רק מטופלים שקבעו תור או שויכו |
| **אחות/ת** | צפייה בתיקים, עדכון מדדים חיוניים, תזמון תורים | אין גישה לאבחנות רגישות |
| **פקיד/ה קבלה** | תזמון תורים, פרטים דמוגרפיים בסיסיים | אין גישה למידע רפואי |
| **מטופל** | צפייה בתיק האישי, תזמון תורים, הורדת מסמכים | רק המידע האישי |

### מדיניות גישה דינמית

- **Context-Aware Access**: בדיקת מיקום, התקן וזמן לפני מתן גישה
- **Session Management**: פקיעת סשן אוטומטית לאחר 15 דקות חוסר פעילות
- **Concurrent Session Control**: הגבלה לסשן פעיל אחד למשתמש

## אבטחת תקשורת

### פרוטוקולי הצפנה

- **TLS 1.3**: תקשורת מוצפנת בין לקוח לשרת
- **Certificate Pinning**: מניעת התקפות Man-in-the-Middle
- **Perfect Forward Secrecy (PFS)**: הגנה על תקשורת עבר גם במקרה של פריצת מפתח

### אבטחת API

- **OAuth 2.0 + JWT**: אימות ואישור מבוסס טוקנים
- **API Rate Limiting**: הגבלת קצב בקשות למניעת התקפות DDoS
- **Input Validation**: בדיקה קפדנית של כל קלט מהמשתמש
- **CORS Policy**: הגבלת גישה רק מדומיינים מורשים

## גיבוי ושחזור

### אסטרטגיית גיבוי

- **Automated Backups**: גיבוי אוטומטי מלא כל 24 שעות
- **Incremental Backups**: גיבויים דיפרנציאליים כל 4 שעות
- **Geographic Redundancy**: שכפול נתונים לפחות 3 אזורי זמינות (Availability Zones)
- **Backup Encryption**: כל הגיבויים מוצפנים ב-AES-256

### תוכנית שחזור אסון (Disaster Recovery)

- **RTO (Recovery Time Objective)**: 4 שעות
- **RPO (Recovery Point Objective)**: 15 דקות
- **Failover Automation**: מעבר אוטומטי למערכת גיבוי במקרה של כשל
- **Regular DR Drills**: תרגילי שחזור רבעוניים

## ניטור ותגובה לאירועים

### מערכת SIEM (Security Information and Event Management)

- **Real-time Monitoring**: ניטור בזמן אמת של פעילות חשודה
- **Anomaly Detection**: זיהוי אוטומטי של דפוסי שימוש חריגים
- **Alert System**: התראות מיידיות למנהלי אבטחה
- **Forensic Analysis**: כלים לחקירה מעמיקה של אירועי אבטחה

### אינדיקטורים למעקב

- ניסיונות כניסה כושלים חוזרים
- גישה למספר רב של תיקי מטופלים בזמן קצר
- ניסיונות גישה מחוץ לשעות עבודה רגילות
- שינויים במידע רגיש ללא הצדקה
- העתקה או הורדה מסיבית של נתונים

## תגובה להפרות אבטחה

### תהליך Breach Response

1. **זיהוי וסיווג** (0-2 שעות)
   - זיהוי היקף ההפרה
   - סיווג רמת חומרה
   - הפעלת צוות תגובה

2. **בידוד והכלה** (2-6 שעות)
   - ניתוק מערכות מושפעות
   - שינוי אישורי גישה
   - איסוף ראיות דיגיטליות

3. **חקירה** (6-48 שעות)
   - ניתוח לוגים ופעילות
   - זיהוי וקטור התקיפה
   - הערכת נזק

4. **תיקון ושחזור** (48-72 שעות)
   - סגירת פרצות אבטחה
   - שחזור מערכות
   - חיזוק אבטחה

5. **דיווח והודעה** (72 שעות)
   - דיווח לרשויות (HHS במקרה של HIPAA)
   - הודעה למטופלים מושפעים
   - תיעוד מלא של האירוע

## ביקורת ותאימות

### ביקורות תקופתיות

- **Internal Audits**: ביקורת פנימית רבעונית
- **External Audits**: ביקורת חיצונית שנתית על ידי גורם מוסמך
- **Penetration Testing**: בדיקות חדירה חצי-שנתיות
- **Vulnerability Scanning**: סריקות אבטחה שבועיות

### תיעוד תאימות

- שמירת תיעוד מלא של כל פעולות האבטחה
- דוחות תאימות תקופתיים להנהלה
- מעקב אחר שינויים בתקנות ועדכון נהלים בהתאם

## הדרכה ומודעות

### תוכנית הדרכה

- **Onboarding Training**: הדרכה מקיפה לעובדים חדשים
- **Annual Refresher**: הדרכת רענון שנתית חובה
- **Phishing Simulations**: תרגילי דיוג תקופתיים
- **Security Awareness**: עדכונים שוטפים על איומים חדשים

## סיכום

מערכת ניהול המרפאה מיישמת גישה מקיפה לאבטחת מידע המשלבת בקרות טכניות, מנהליות ופיזיות. העמידה בתקני HIPAA והגנה על פרטיות המטופלים הם בראש סדר העדיפויות, עם ניטור מתמיד ושיפור מתמשך של אמצעי האבטחה.
